主な機能

リアルタイム監視からスケジュール管理まで、充実した機能を搭載

1

🎥 リアルタイム監視

YOLOv11を使用したWebカメラ映像のリアルタイム物体検知。人物とスマホを自動認識し、バウンディングボックスで可視化します。検知結果はSQLiteに記録され、後から統計分析が可能です。

2

🎤 音声入力スケジュール登録

Whisper（STT）とLLMを組み合わせた音声入力機能。「今日の15時に会議」と話すだけで、日時・タイトル・説明を自動抽出してスケジュール登録できます。音声チャット機能も搭載。

3

🚶 一時的な不在許可（PASS）

昼休憩や買い出しなど、一時的に監視を止めたい時にプリセットまたはカスタム時間で不在検知を一時停止。許可中の不在は統計・アラート・Slackレポートから除外されます。

4

📊 Slack定期レポート

毎日指定時刻に検知統計・前面アプリ滞在時間・システムイベントをSlackに自動投稿。「当日累積」と「直近送信以降の差分」の2セクションを同梱し、業務実態を可視化します。

5

⌨️ タイピング監視

グローバルキーフック（uiohook-napi）で1分単位の入力数・最長連続入力時間を記録。ダッシュボードとSlackレポートで生産性を可視化できます。

6

📈 検知ダッシュボード

期間・集計単位・イベント種別でフィルタリング可能な統計ダッシュボード。検知ログ・アラート件数・前面アプリ滞在時間を時系列グラフで表示し、CSV エクスポートにも対応。

7

💬 音声チャット

STT→LLM→VOICEVOXのパイプラインで、キャラクター音声による対話が可能。会話履歴はローカルに保存され、ストリーミングで途中生成も表示します。

セットアップガイド

インストールから初期設定まで、ステップバイステップで解説

1 必要環境の確認

以下の環境が必要です:

  • Node.js 20 以上（node-llama-cpp の要件）
  • macOS 13 以上を推奨（一部機能がmacOS依存）
  • Xcode Command Line Tools / CMake / pkg-config
  • ffmpeg（音声形式変換に使用）
  • VOICEVOX エンジン（オプション: 読み上げ機能）
  • Slack Incoming Webhook URL（オプション: Slack連携）

2 基本セットアップ

リポジトリをクローンして依存関係をインストールします:

git clone https://github.com/tmys-sasaki/kanshichan.git cd kanshichan npm install

YOLOモデルを models/yolo11n.onnx として配置してください。

3 音声入力機能のセットアップ（オプション）

必要ツールのインストール:

# macOS の場合 brew install cmake ffmpeg

モデルファイルの配置:

Whisper モデル (ggml-base.bin) と LLM モデル (swallow-8b-v0.5-q4.gguf) を models/ ディレクトリに配置してください。

# Whisper モデルのダウンロード cd models curl -L https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-base.bin -o ggml-base.bin

whisper-cli のビルド:

git clone https://github.com/ggerganov/whisper.cpp.git cd whisper.cpp cmake -B build -DCMAKE_BUILD_TYPE=Release -DWHISPER_BUILD_EXAMPLES=ON cmake --build build --config Release --target whisper-cli -j 4
⚠️ 注意: whisper-cli へのパスを環境変数 WHISPER_CLI_PATH で指定するか、PATHに追加してください。

4 アプリケーションの起動

npm start

起動時に macOS のカメラ許可ダイアログが表示されます。許可後、監視が自動開始されます。

5 タイピング監視の有効化（オプション）

npm install uiohook-napi --build-from-source \ --runtime=electron --target=38.2.0 \ --disturl=https://electronjs.org/headers npx electron-rebuild -f -w uiohook-napi

macOS の「システム設定 > プライバシーとセキュリティ > アクセシビリティ」でアプリを追加し、チェックを入れてください。

よくある問題と解決方法

「モデルが見つかりません」エラー:

models/ggml-base.bin と models/swallow-8b-v0.5-q4.gguf が正しく配置されているか確認してください。

「whisper-cli executable not found」エラー:

whisper.cpp のビルドに失敗しているか、CLI へのパスが通っていない可能性があります。

「音声ファイルの変換に失敗しました」エラー:

ffmpeg がインストールされているか確認: which ffmpeg

読み上げリクエストはエラー扱いとなり、UI 側にはアラートが表示されません（ログのみに記録）。VOICEVOX エンジンを起動してから再度お試しください。

ブラウザ/Electron の通知権限を確認してください。macOS の「システム設定 > 通知」でアプリの通知が許可されているか確認してください。

アーキテクチャ概要

監視ちゃんは以下のレイヤーで構成されています:

  • メインプロセス: YOLO検知、IPC通信、システムイベント監視
  • レンダラプロセス: UI制御、カメラストリーム、スケジュール管理
  • 音声処理: Whisper（STT）→ LLM（抽出）→ VOICEVOX（TTS）のパイプライン
  • データ永続化: SQLite（検知ログ、前面アプリ、不在許可履歴）、localStorage（設定・スケジュール）
